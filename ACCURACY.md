## Self-Correction Demonstration

A self-correction sequence is a sequence of tool invocations where the agent's first attempt produced an error or an unexpected output and the agent re-planned. SWORN logs each as `{first_attempt_invocation_id, error_observed, replanning_rationale, second_attempt_invocation_id}`. The eval harness emits a count of self-correction sequences per case and the proportion of them that ultimately produced a corroborated finding.

## Reproduction

Every number above is regenerable from the corpus + the SWORN binary at a fixed git SHA. The report writes its own `report_metadata.json` with:
- Git SHA of SWORN
- `pip freeze` of the venv
- Corpus image SHA-256s
- LLM provider and model version
- Wall-clock duration per case

If a number cannot be reproduced from those inputs, that is the bug.
